Resume Project Reflection

The skeleton framework is useful when creating applications, websites, or projects with minimal use of coding.
Main benefits of using this framework include the short usage of code, increased productivity because the code and
be used over and over again with minimum changes required. The main weakness associated with the skeleton framework
include organizing the grid in a way that makes a project look the way that a developer wants. When comparing with
the portfolio project using media queries, skeleton was harder to a certain extent. The reason for this notion is
that the CSS had to be coded using a grid system, whereas for the portfolio a grid was not specifically utilized.  

Flexbox Project Reflection

Flexbox grid system provides developers with a column and row type of structure to operate with. The main advantage
of this framework is that it resizes when you shrink or expand the webpage window. It is also useful for positioning and
aligning material on a webpage. Weaknesses associated with flexbox include it being not supported by all or some browsers
such as internet explorer. When compared to the previous responsive systems used in this course, flexbox is comparable to
Skeleton in terms of operation, usefulness, and programming. However, it is “harder” to use than the portfolio
homepage in terms of media queries and coding.  

Bootstrap Project Reflection

The bootstrap framework allows developers with the ability to create forms, buttons, and other navigational materials for webpage and
interface design. This project was completed using an existing external CSS code that is linked to the main webapage.
This project was slighlty harder to complete than the previous projects, however in terms of speed it is slower than the flexbox
and the resume assignments.
